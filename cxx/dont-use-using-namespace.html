Don't use using namespace

5.3.2022

Each programming tutorial starts with the famous "Hello World!" example. If you type "c++ hello world" into the Google you will probably find something like the following code: 

#include <iostream>

using namespace std;

int main()
{
    cout << "Hello, World!" << endl;
    return 0;
}

If you declare using namespace std; you can use functions, classes and objects from the std namespace without specifying std:: before a name. However, this means that you can't use the same name for your class or function. Even worse, if you import two namespaces and both of them happens to contain a function with the same name a call to the function is ambiguous and you need to disambiguate it by using the namespace prefix anyway. Last but not least, and this is more of an opinion than a best practice, the code looks less C++-ish without namespace prefixes. If you look at any serious C++ codebase, the code will probably contain the namespace prefixes. It makes a code more readable since you immediately see from where a given class or function comes from.

std:: boost:: nested std::chrono:: std::filesystem:: or std::ranges::views.
